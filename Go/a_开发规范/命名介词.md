# 命名中的介词使用规范

> 编程命名中的英文介词决定了 API 是否"读起来像自然语言"。本文整理 Go（以及通用编程语境）中最常见的介词及其惯用场景，覆盖 **函数名 / 方法名 / 字段名** 中"动作 + 对象 + 关系"的标准搭配，帮助写出对调用方友好的 API。

## 1. 总览

| 介词       | 核心语义              | 典型搭配                        | 典型场景                |
| -------- | ----------------- | --------------------------- | ------------------- |
| `to`     | 转换为 / 面向目标         | `XToY` / `WriteTo` / `ApplyTo` | 类型转换、写入目标、应用变更      |
| `of`     | "...的" / 所属        | `IndexOf` / `ListOf` / `TypeOf` | 取属性、由参数构造、计数        |
| `by`     | 通过 / 按某键          | `FindByID` / `SortByAge`     | DAO 查询、排序、分组        |
| `from`   | 来自 / 反向构造          | `FromJSON` / `ReadFrom`     | 反序列化、读取源、派生         |
| `with`   | 附加 / 配置 / 包裹       | `WithCancel` / `WithLogger`  | functional options、不可变派生 |
| `in`     | 在容器/范围中 / 就地      | `InRange` / `SortInPlace`   | 集合判定、原地修改           |
| `on`     | 基于 / 事件钩子         | `OnClick` / `OnError`       | 回调、事件               |
| `at`     | 位置 / 时间点          | `CreatedAt` / `CharAt`       | 时间戳字段、索引访问          |
| `for`    | 服务于谁 / 为了目的       | `WaitFor` / `HandlerFor`    | 等待条件、面向某对象的服务       |
| `as`     | 视作 / 换视角          | `errors.As` / `AsReader`     | 类型断言、视图转换           |
| `into`   | 灌入已存在的容器          | `DecodeInto(&v)`            | 解码到指针、写入到 buffer    |
| `via`    | 经由中介              | `SendVia(proxy)`            | 网络中转、间接路径           |
| `per`    | 每单位               | `RequestsPerSecond`         | 速率、单价等比率字段          |
| `between` | 二元区间             | `DurationBetween(a,b)`      | 区间计算                |
| `without` | 排除某项 / 不带         | `context.WithoutCancel`     | 取消某种属性、过滤集合         |

## 2. 高频介词：to / of / by / from / with

### 2.1 `to` —— 转换为 / 面向某个目标

**语义**：把 A 变成 B，或把动作的方向指向某个目标。

| 用法     | 典型命名                            | 场景             |
| ------ | ------------------------------- | -------------- |
| 类型转换   | `ToString` / `ToInt` / `ToJSON` | 一个形式 → 另一个形式   |
| 写入到目标  | `WriteTo(w io.Writer)` / `CopyTo(dst)` | 把数据"写到"某处      |
| 单位转换   | `MetersToFeet` / `UTCToLocal`   | 跨域转换           |
| 应用到对象  | `ApplyTo(cfg *Config)`          | 作用于目标          |

```go
func (u User) ToDTO() UserDTO                          // 模型 → DTO
func (b *Buffer) WriteTo(w io.Writer) (int64, error)   // 标准库 io.WriterTo
func StringToInt(s string) (int, error)
```

> 经验：**"X 转 Y"用 `XToY`**，不要写成 `ConvertX`、`TransformXToY`，前者更短、更可组合。

---

### 2.2 `of` —— "...的" / 所属关系

**语义**：表示"属于、来源于、由...构成"，常用于 **构造、查询、计数**。

| 用法      | 典型命名                                | 场景               |
| ------- | ----------------------------------- | ---------------- |
| 工厂构造    | `ListOf(1,2,3)` / `MapOf(k,v)` / `ArrayOf[T]` | 由参数构造一个集合        |
| 计数 / 长度 | `LengthOf(s)` / `SizeOf(x)`         | "...的数量 / 长度"    |
| 取属性     | `IndexOf(x)` / `KeyOf(v)` / `TypeOf(v)` | "...的索引 / 键 / 类型" |
| 实例归属    | `InstanceOf` / `KindOf`             | 类型判断             |

```go
reflect.TypeOf(x)          // x 的类型
reflect.ValueOf(x)         // x 的值
slices.Index(s, x)         // Go 标准库更倾向直接动词，少见 IndexOf
```

> 经验：`of` 通常出现在 **"取某物的某属性"** 或 **"由参数生成集合"** 时。Go 风格更偏短动词，`of` 多见于库 API 边界。

---

### 2.3 `by` —— 通过 / 按照某种方式

**语义**：表示"按某个键/规则/手段来执行动作"。

| 用法      | 典型命名                              | 场景                    |
| ------- | --------------------------------- | --------------------- |
| 按字段查询   | `FindByID` / `GetByName` / `QueryByEmail` | DAO / Repository 经典命名 |
| 按字段排序分组 | `SortByAge` / `GroupByCategory` / `OrderByCreatedAt` | 排序、分组                 |
| 按方式执行   | `LoginByToken` / `AuthByOAuth`    | 通过某种手段                |
| 排序键     | `SortBy(func(a,b) int)`           | 函数式排序                 |

```go
func (r *UserRepo) FindByID(id int64) (*User, error)
func (r *UserRepo) FindByEmail(email string) (*User, error)
slices.SortFunc(users, func(a, b User) int { ... })  // Go 偏向 Func 后缀
```

> 经验：`By` 后面**几乎一定接的是"键 / 维度"**（ID、Name、Status…），不是动作本身。

---

### 2.4 `from` —— 来自 / 从...构造

**语义**：和 `to` 互为镜像，强调**来源**。

| 用法       | 典型命名                                  | 场景                |
| -------- | ------------------------------------- | ----------------- |
| 从某种格式构造  | `ParseFromJSON` / `FromBytes` / `UserFromRow` | 反序列化 / 反向构造       |
| 从源读取     | `ReadFrom(r io.Reader)`               | 标准库 `io.ReaderFrom` |
| 派生       | `DeriveKeyFromPassword`               | 从某物派生             |

```go
func (b *Buffer) ReadFrom(r io.Reader) (int64, error)
func UserFromRow(row *sql.Row) (*User, error)
```

> 经验：**`FromX` 常作为 `ToX` 的对立面**。
> - 输入是"原料"用 `From`：`UserFromJSON(data)`；
> - 输入是"参数"用 `Of` / `New`：`NewUser(name, age)` / `ListOf(1,2,3)`。

---

### 2.5 `with` —— 携带 / 附加 / 配置

**语义**：在已有对象上**附加一项额外信息**，返回新对象（不可变风格），或包裹一段执行上下文。

| 用法                  | 典型命名                                          | 场景               |
| ------------------- | --------------------------------------------- | ---------------- |
| 派生新 context / config | `context.WithCancel`, `WithTimeout`, `WithValue` | Go 经典模式          |
| 配置项 / Builder       | `WithLogger(l)` / `WithRetries(3)`            | Functional Options |
| 加锁 / 资源包裹           | `WithLock(mu, fn)` / `WithTx(db, fn)`         | 高阶函数包装           |

```go
ctx, cancel := context.WithTimeout(ctx, 3*time.Second)
client := http.NewClient(WithTimeout(5*time.Second), WithRetries(3))
```

> 经验：**`With` 一定意味着"返回新值 / 不可变" 或 "包裹执行"**，**不要用它做 setter**。
> 反例：`user.WithName("alice")` 如果是就地修改，应改名为 `SetName`。

## 3. 次高频介词：in / on / at / for / as

### 3.1 `in` —— 在...之中 / 就地

| 用法       | 典型命名                            | 场景               |
| -------- | ------------------------------- | ---------------- |
| 就地（原地修改） | `SortInPlace` / `ReverseInPlace` | 区分"返回新值" vs "原地" |
| 在某容器中    | `IsIn(set)` / `ExistsIn`        | 集合判定（Go 直接 `Contains`） |
| 在某领域内    | `InRange(lo, hi)` / `InTransaction(fn)` | 表示作用域            |

```go
slices.Contains(s, x)             // Go 不爱用 IsIn，直接 Contains
func (db *DB) InTransaction(fn func(tx *Tx) error) error
```

---

### 3.2 `on` —— 基于 / 事件触发

| 用法      | 典型命名                            | 场景            |
| ------- | ------------------------------- | ------------- |
| 事件回调    | `OnClick`, `OnError`, `OnClose` | UI / 事件钩子     |
| 在某对象上执行 | `LockOn(resource)`              | 较少见           |
| 依赖某物    | `DependsOn`                     | DAG / 依赖关系    |

```go
server.OnShutdown(func() { ... })
btn.OnClick = handler
```

> `on` 几乎是**事件 / 回调专属**。看到 `OnXxx`，第一反应应该是钩子。

---

### 3.3 `at` —— 在某个位置 / 时间点

| 用法     | 典型命名                                       | 场景                |
| ------ | ------------------------------------------ | ----------------- |
| 按下标访问  | `At(i)` / `CharAt(i)` / `ElementAt`        | 索引访问（Go 直接 `s[i]`） |
| 时间点字段  | `CreatedAt`, `UpdatedAt`, `DeletedAt`, `ExpiresAt` | 数据库 / 模型字段惯例      |
| 在某地点   | `LookupAt(addr)`                           | 较少                |

```go
type User struct {
    CreatedAt time.Time
    UpdatedAt time.Time
    DeletedAt *time.Time
}
```

> 经验：**所有"时间戳"字段都用 `XxxAt`**，跨语言通用。
> 不要写 `CreateTime` / `CreationDate` / `CreateDt`（中式英语或冗余命名）。

---

### 3.4 `for` —— 用于 / 为了某目的

| 用法           | 典型命名                              | 场景            |
| ------------ | --------------------------------- | ------------- |
| 检索 / 查找面向某对象 | `LookupFor(user)` / `PriceFor(product)` | "为...而..."的服务 |
| 等待 / 监听      | `WaitFor(cond)` / `ListenFor(event)` | 表示目的          |
| 分支变体         | `HandlerFor(method)`              | 根据上下文返回       |

```go
func WaitFor(cond func() bool, timeout time.Duration) error
func HandlerFor(method string) http.Handler
```

> 经验：和 `By` 容易混淆。
> - **`By` = 用什么来定位**：`FindByID(id)`；
> - **`For` = 服务于谁 / 什么目的**：`PermissionsFor(user)`。

---

### 3.5 `as` —— 视作 / 转换视角

| 用法           | 典型命名                                                  | 场景              |
| ------------ | ----------------------------------------------------- | --------------- |
| 类型断言 / 视角转换  | `errors.As(err, &target)` / `AsReader()` / `AsSlice()` | 同一对象不同视角        |
| 角色化          | `RunAsRoot()` / `LoginAsAdmin()`                      | 以某身份执行          |

```go
var pathErr *os.PathError
if errors.As(err, &pathErr) { ... }
func (m Map) AsSlice() []Entry
```

> 与 `to` 的差异：**`As` 通常不复制数据，只是换个壳/视角**；**`To` 通常构造新值**。

## 4. 其他常见介词

### 4.1 `into` —— 灌入已有容器

```go
json.Unmarshal(data, &v)                         // 标准库直接动词
func DecodeInto(r io.Reader, dst any) error     // 强调写入到已存在容器
```

`into` 强调 **目标容器已经存在**，区别于 `to`（构造新值）。

---

### 4.2 `via` —— 经由 / 通过中介

不算高频，但在网络 / 路由中常见：

```go
func SendVia(proxy Proxy, msg Message) error
```

`via` 比 `by` 更强调"经过中转 / 媒介"。

---

### 4.3 `per` —— 每 / 每个

```go
RequestsPerSecond
CostPerUnit
BytesPerPacket
```

主要用在 **比率 / 速率 / 单价** 字段，**不用于函数名**。

---

### 4.4 `between` / `among` —— 在...之间

```go
func DurationBetween(a, b time.Time) time.Duration
func ChooseAmong(opts []Option) Option   // 三者及以上
```

`between` 用于二元；`among` 用于三元及以上（实际开发中 `among` 几乎不用，多用 `from`）。

---

### 4.5 `without` —— 不带 / 排除

```go
ctx := context.WithoutCancel(parent)   // Go 1.21+
func ListUsersWithoutAdmin() []User
```

和 `with` 是镜像，使用频率低很多。

---

## 5. 容易混淆的对比

| 对比               | 区分要点                       | 例子                                    |
| ---------------- | -------------------------- | ------------------------------------- |
| `By` vs `For`    | By = 按什么键找；For = 服务于谁      | `FindByID` vs `PermissionsFor(user)`  |
| `Of` vs `From`   | Of = 由参数构造；From = 由源解析 / 派生 | `ListOf(1,2,3)` vs `UserFromJSON(b)`  |
| `To` vs `As`     | To = 产生新值；As = 同一对象换视角     | `ToString()` vs `AsReader()`          |
| `To` vs `Into`   | To = 返回新值；Into = 写入已存在的容器   | `ToSlice()` vs `DecodeInto(&v)`       |
| `In` vs `On`     | In = 在容器 / 范围中；On = 事件钩子   | `InRange` vs `OnClick`                |
| `With` vs `Set`  | With = 返回新值；Set = 就地修改     | `ctx.WithValue(...)` vs `u.SetName(...)` |

---

## 6. 命名经验法则

1. **介词只用一个**
   `GetUserByIDFromCacheWithLock` 这种串烧是反模式，应**拆函数**或**改用 functional options**。

2. **对称命名要成对**
   - 有 `ToX` 就考虑提供 `FromX`；
   - 有 `WithX` 就考虑提供 `WithoutX`；
   - 有 `Encode` 就考虑提供 `Decode`。

3. **Go 语境下慎用介词**
   Go 风格倾向**短动词**（`Contains`、`Index`、`Sort`），介词命名多见于库 API 边界（`io.WriterTo`、`context.WithCancel`、`errors.As`）。**业务代码内部不必滥用**。

4. **三条跨语言共识**
   - 时间戳一律 `XxxAt`：`CreatedAt`、`ExpiresAt`；
   - 回调一律 `OnXxx`：`OnClick`、`OnError`；
   - 查询一律 `FindByXxx` / `GetByXxx`：`FindByID`、`GetByEmail`。

5. **介词不是修饰符堆栈**
   如果你发现需要三个以上介词才能表达一个函数的意图，多半是**函数职责过重**，该拆了。

6. **优先看标准库怎么写**
   Go 标准库是介词命名的最佳教科书：
   - `io.WriterTo` / `io.ReaderFrom`（成对）；
   - `context.WithCancel` / `WithTimeout` / `WithValue` / `WithoutCancel`（不可变派生）；
   - `errors.As` / `errors.Is` / `errors.Unwrap`（视角与判定）；
   - `reflect.TypeOf` / `ValueOf`（取属性）。
